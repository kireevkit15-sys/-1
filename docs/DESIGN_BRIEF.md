# Техническое задание для веб-дизайнера

## Платформа РАЗУМ

**Дата:** 2026-04-07
**Заказчик:** команда РАЗУМ
**Формат:** PWA (Progressive Web App), mobile-first + десктоп-адаптив
**Результат:** UI-kit / макеты всех экранов для внедрения в код

---

## 1. О проекте

**РАЗУМ** -- интеллектуальная PWA-платформа для мужчин 18-45 лет. Ключевая механика -- RPG-батлы знаний в формате 1v1 "Осада крепости": игроки по очереди атакуют и защищаются, отвечая на вопросы разной сложности. Помимо баттлов есть система обучения, AI-наставник (сократический метод) и персональный профиль-аватар с 5 характеристиками.

**Позиционирование:** серьёзная интеллектуальная платформа с игровой механикой. Не казуальная викторина, а инструмент прокачки мышления с соревновательным элементом.

**Целевая аудитория:** мужчины 18-45 лет, интересующиеся саморазвитием, стратегическим мышлением, логикой. Технически грамотные (Telegram как основной вход).

## 2. Текущее состояние

Приложение уже разработано и функционирует. Ссылка на действующий сервер будет предоставлена отдельно для ознакомления с текущим состоянием.

Нужен визуальный редизайн. Дизайнер **свободен** в:
- Визуальном стиле, цветах, эффектах, иконках
- Расположении элементов на экране
- Стилистике карточек, кнопок, навигации
- Выборе шрифтов и типографики

Дизайнер **должен сохранить**:
- Набор экранов и их назначение (что на каком экране)
- Данные, которые отображаются (статы, счёт, HP, XP и т.д.)
- Бизнес-логику (5 фаз баттла, 3 уровня сложности, 5 характеристик)
- 4 вкладки навигации (Главная, Баттл, Обучение, Профиль)

### 2.1 Экраны приложения

| Экран | Описание |
|-------|----------|
| Логин | Вход через Telegram + email/пароль |
| Главная | Стрик, ежедневная разминка, кнопка баттла, мини-статистика, XP-прогресс |
| Поиск баттла | Выбор режима (бот/человек), анимация поиска, экран "VS" перед стартом |
| Баттл | 5 фаз: выбор категории -> атака -> защита -> результат раунда -> финал |
| Обучение | Список модулей по 2 веткам (Стратегия, Логика), прогресс, разблокировка |
| Модуль обучения | Вопросы модуля + кнопка вызова AI-наставника |
| AI-наставник | Чат с AI в контексте изучаемой темы (сократический метод) |
| Профиль | Аватар, диаграмма 5 статов, XP, стрик, история баттлов |
| Лидерборд | Топ-20 игроков |
| Разминка | 5 ежедневных вопросов + стрик |

### 2.2 Навигация

**Мобилка:** нижняя панель -- 4 вкладки:
1. **Главная**
2. **Баттл**
3. **Обучение**
4. **Профиль**

**Десктоп:** боковая или верхняя навигация (на усмотрение дизайнера), те же 4 пункта.

### 2.3 Текущая палитра

Сейчас используется тёмная тёплая палитра ("Dark Academia"): тёмные сине-серые фоны, медный акцент, бронза/серебро/золото для уровней сложности, тёплый бежевый текст.

Дизайнер может предложить **полностью новую палитру** или доработать текущую. Требуется две темы:
- **Тёмная** (основная, дефолт)
- **Светлая** (полноценная альтернатива с переключением, не просто инвертированная)

## 3. User Flow (путь пользователя)

### 3.1 Новый пользователь (первый вход)

```
Логин
  -> Вход через Telegram (или email-регистрация)
  -> Главная (первое знакомство с платформой)
     Видит: пустую статистику (0 баттлов, 0 побед), XP на нуле,
     приглашение к действию -- "Начни разминку" или "Попробуй баттл"
  -> Разминка (5 вопросов) -- первое касание с контентом
  -> Возврат на Главную (видит: стрик = 1, заработанный XP)
  -> Баттл с ботом (тренировка без рейтинга)
     Поиск -> VS экран -> 5 фаз баттла -> Финал с наградами
  -> Профиль (видит: первые статы на диаграмме, уровень 1)
```

### 3.2 Возвращающийся пользователь (ежедневный цикл)

```
Главная
  -> Видит: стрик (N дней подряд), текущий уровень, рейтинг
  -> Разминка (утренняя тренировка, 5 вопросов) -- поддержать стрик
  -> Баттл (1v1 с человеком или ботом) -- основная активность
     Можно сыграть несколько подряд ("Ещё баттл" на финальном экране)
  -> Обучение -- прокачка слабых навыков
     Список модулей -> Выбор модуля -> Вопросы модуля
     -> AI-наставник (чат: "Не понял тему -- спроси AI")
  -> Профиль -- посмотреть прогресс, диаграмму статов
  -> Лидерборд -- сравнить себя с другими
```

### 3.3 Ключевые переходы

| Откуда | Куда | Когда |
|--------|------|-------|
| Главная | Разминка | Тап на карточку разминки |
| Главная | Поиск баттла | Тап на карточку баттла |
| Поиск баттла | Баттл | Соперник найден, после экрана VS |
| Баттл (финал) | Поиск баттла | Кнопка "Ещё баттл" |
| Баттл (финал) | Главная | Кнопка "На главную" |
| Обучение | Модуль | Тап на модуль |
| Модуль | AI-наставник | Кнопка "Спросить AI" |
| AI-наставник | Модуль | Назад |
| Любой экран | Любая вкладка | Через нижнюю навигацию |

## 4. Что нужно от дизайнера

### 4.1 Общий визуальный стиль

Нужен целостный визуальный язык платформы, который передаёт:

- **Интеллект и глубину** -- это не казуальная игра, а серьёзная платформа
- **Соревновательный дух** -- RPG/киберспорт эстетика, напряжение баттла
- **Мужской характер** -- без "милого", без пастели, без roundness ради roundness
- **Премиальность** -- ощущение качественного продукта, не дешёвого приложения

**Референсы направлений** (не копировать, а вдохновляться):
- Игровые лобби (Valorant, League of Legends, Chess.com)
- Фитнес-трекеры премиум-сегмента (Whoop, Oura)
- Финтех dark mode (Revolut, Robinhood dark)
- Образовательные платформы (Duolingo -- механики, не стиль)

### 4.2 Требуемые артефакты

1. **Стайл-гайд / UI-kit:**
   - Палитра цветов для **обеих тем** (dark + light): основные, акценты, семантические
   - Типографика (шрифты, размеры, веса для заголовков/текста/подписей)
   - Сетка и отступы
   - Радиусы скруглений
   - Тени и эффекты -- для обеих тем
   - Иконки (стиль, размеры)
   - Переключатель тем (toggle dark/light) -- расположение и вид

2. **Компоненты:**
   - Кнопки (primary, secondary, danger, disabled)
   - Карточки (обычная, акцентная, с бордером)
   - Инпуты (обычный, focus, error)
   - Навигационная панель (BottomNav для mobile, боковая/верхняя для desktop)
   - Прогресс-бары (XP, здоровье)
   - Бейджи (уровень, рейтинг, класс мыслителя)
   - Таймер (60 сек)
   - Модалки / тосты
   - Чат-интерфейс (сообщения пользователя и AI)

3. **Макеты всех экранов в двух темах (dark + light):**
   - Mobile (375px) -- основной размер
   - Desktop (1280px+) -- адаптивная версия
   - Все экраны из таблицы 2.1
   - У баттла: все 5 фаз (отдельные фреймы)
   - У поиска баттла: все 3 состояния (выбор, поиск, VS)
   - Основные состояния: loading (загрузка данных), empty (нет данных)

## 5. Детальное описание экранов

### 5.1 Логин

**Функционал:**
- Логотип "РАЗУМ" + слоган
- Кнопка "Войти через Telegram" (основная, крупная)
- Разделитель "или"
- Форма email + пароль
- Переключатель "Войти / Зарегистрироваться"
- Юридический текст внизу

**Ощущение:** строгий, premium вход. Не перегружен.

### 5.2 Главная

**Функционал:**
- Шапка: логотип "РАЗУМ" + счётчик стрика (серия дней подряд)
- Карточка ежедневной разминки (CTA: "Начать разминку", 5 вопросов)
- Карточка баттла (CTA: "Найти соперника")
- Блок статистики: баттлы / победы / винрейт / рейтинг
- XP прогресс-бар до следующего уровня

**Empty state:** новый пользователь видит нулевую статистику и мотивирующие CTA.

**Ощущение:** дашборд игрока. Быстрый доступ к действиям. Мотивация через числа.

### 5.3 Поиск баттла

**3 последовательных состояния:**

**A. Выбор режима:**
- Карточка "Играть с ботом" (тренировка без рейтинга)
- Карточка "Найти соперника" (рейтинговый матч)
- Блок с правилами баттла

**B. Поиск соперника:**
- Анимация ожидания
- Текст "Ищем соперника..."
- Кнопка "Отмена"

**C. Соперник найден (экран VS):**
- Два игрока друг напротив друга (аватары + имена + рейтинги)
- Визуальное противостояние ("VS")
- Индикатор загрузки перед стартом

**Ощущение:** нарастающее напряжение. От спокойного выбора -> к ожиданию -> к драматичному VS.

### 5.4 Баттл -- КЛЮЧЕВОЙ ЭКРАН

Самый важный экран платформы. Состоит из 5 фаз, каждая требует отдельного дизайна.

**Верхняя панель (ScoreBar)** -- присутствует во всех фазах:
- Игрок 1 (слева): аватар + полоска здоровья (HP)
- Центр: счёт + номер раунда
- Игрок 2 (справа): аватар + полоска здоровья (HP)
- HP убывает при получении урона

**Фаза 1: ВЫБОР КАТЕГОРИИ**
- 2-3 варианта категорий (Стратегия, Логика и т.д.)
- Атакующий игрок выбирает тему для раунда

**Фаза 2: АТАКА** (основная игровая фаза)
- **Таймер** -- 60 секунд на ответ, визуально показывает оставшееся время
- **Выбор сложности** -- 3 уровня (см. раздел 6):
  - Лёгкий (10 урона)
  - Средний (20 урона)
  - Сложный (30-35 урона)
- **Вопрос** -- текст + 4 варианта ответа

**Фаза 3: ЗАЩИТА**
- 3 варианта действия для защищающегося:
  - "Принять" -- смирение с результатом
  - "Оспорить" -- попытка отменить атаку
  - "Контратака" -- ответная атака
- Каждый вариант отличается риском и наградой

**Фаза 4: РЕЗУЛЬТАТ РАУНДА**
- Отображение нанесённого/полученного урона (или промаха)
- Статы раунда: очки, точность, результат защиты

**Фаза 5: ФИНАЛ БАТТЛА**
- Результат: победа / поражение / ничья (визуально разные)
- Итоговый счёт
- Награды: полученный XP, изменение рейтинга (+/-)
- Кнопки: "Ещё баттл" / "На главную"

**Ощущение:** динамика, напряжение, удовлетворение. Как шахматный матч с элементами файтинга.

### 5.5 Обучение (список модулей)

**Функционал:**
- 2 ветви: Стратегия и Логика
- Каждая ветвь -- группа модулей
- Модуль: название + прогресс
- Состояния модуля: доступен / в процессе / завершён / заблокирован

**Ощущение:** древо прокачки как в RPG. Чувство прогресса и разблокировки.

### 5.6 Модуль обучения (внутри модуля)

**Функционал:**
- Название модуля и описание темы
- Вопросы модуля (последовательно, как в разминке)
- Прогресс внутри модуля (3/5 вопросов)
- Объяснение после каждого ответа
- Кнопка "Спросить AI-наставника" -- переход в чат по текущей теме

**Ощущение:** обучение с глубиной. Не просто вопрос-ответ, а понимание.

### 5.7 AI-наставник (чат)

**Функционал:**
- Чат-интерфейс: сообщения пользователя (справа) и AI (слева)
- Контекст: тема текущего модуля (отображается вверху)
- Поле ввода сообщения
- Индикатор "AI печатает..."
- Лимит: 1 диалог в день (бесплатно), до 10 обменов в диалоге
- Список прошлых диалогов (тема, дата, количество сообщений)

AI ведёт диалог сократическим методом -- не даёт готовых ответов, а задаёт наводящие вопросы, помогая пользователю самому прийти к пониманию.

**Ощущение:** разговор с умным наставником. Не чат-бот, а персональный тьютор.

### 5.8 Профиль

**Функционал:**
- Аватар + имя + бейджи (уровень, рейтинг, класс мыслителя)
- XP прогресс-бар до следующего уровня
- Статистика: всего баттлов / побед / винрейт
- Серия дней (стрик)
- **Диаграмма 5 характеристик** (паутинная/радарная):
  - Стратегия, Логика, Эрудиция, Риторика, Интуиция
- XP по каждому навыку
- История последних 10 баттлов (противник, счёт, W/L)
- Переключатель тем (dark/light) -- здесь или в настройках

**Ощущение:** RPG-экран персонажа. Гордость за прогресс. Мотивация расти.

### 5.9 Лидерборд

**Функционал:**
- Табы: Неделя / Месяц / Всё время
- Топ-3 с визуальным выделением
- Список 4-20 места
- Позиция текущего игрока (фиксирована внизу если не в топ-20)
- Кнопка "Поделиться профилем"

**Ощущение:** спортивная таблица. Соревновательная мотивация.

### 5.10 Разминка

**Функционал:**
- Прогресс: "Вопрос 1/5", "2/5", ...
- Карточка вопроса + 4 варианта ответа
- Таймер
- Обратная связь после ответа: правильно/неправильно + объяснение
- Финальный экран: результат (4/5), стрик, полученный XP

**Ощущение:** быстрая утренняя тренировка. Бодрит.

## 6. Система сложности (3 уровня)

В баттле игрок выбирает сложность вопроса -- чем сложнее, тем больше урон при правильном ответе, но выше риск ошибки.

| Уровень | Название | Урон при правильном ответе | Ощущение |
|---------|----------|---------------------------|----------|
| 1 | Лёгкий | 10 | Безопасный, базовый |
| 2 | Средний | 20 | Серьёзный, сбалансированный |
| 3 | Сложный | 30-35 | Элитный, высокая ставка |

Три уровня должны **визуально чётко отличаться** друг от друга. Конкретные решения (иконки, цвета, эффекты) -- на усмотрение дизайнера. Важно, чтобы сложный уровень ощущался как "риск ради большой награды".

## 7. Система характеристик аватара

5 характеристик (статов) -- ключ к RPG-идентичности:

| Стат | Описание | Тематика вопросов |
|------|----------|-------------------|
| Стратегия | Планирование, принятие решений | Бизнес, лидерство, переговоры |
| Логика | Анализ, дедукция | Формальная логика, критическое мышление |
| Эрудиция | Широкие знания | Наука, история, культура |
| Риторика | Убеждение, аргументация | Дебаты, софистика, ораторство |
| Интуиция | Быстрые решения, паттерны | Вероятности, статистика, оценка |

**Класс мыслителя** -- определяется доминирующим статом:
- Стратег, Логик, Эрудит, Ритор, Интуит

Отображается на профиле как бейдж/лейбл. Визуальное решение -- на усмотрение дизайнера.

## 8. Анимации и микровзаимодействия

Дизайнер определяет визуальный стиль анимаций. Ниже -- список моментов, где анимация обязательна:

- **Появление элементов** -- карточки, кнопки, результаты появляются не мгновенно
- **Нажатие кнопок** -- тактильная обратная связь
- **Урон в баттле** -- акцентная анимация при нанесении/получении урона
- **Победа/поражение** -- финальный экран с выразительным эффектом
- **Поиск соперника** -- состояние ожидания
- **HP-бары** -- плавное уменьшение
- **Таймер** -- визуальное нарастание срочности при малом времени
- **Прогресс-бары** -- плавное заполнение

Общий принцип: анимации функциональные, не декоративные. Должны усиливать ощущения, а не отвлекать.

## 9. Адаптивность и технические ограничения

### Mobile (основной)
- **Основной размер:** 375px (iPhone SE), тестировать на 375/390/414px
- **Ориентация:** portrait
- **Safe area:** учитывать вырезы (iPhone notch) и home indicator
- **Навигация:** нижняя панель (4 вкладки)

### Desktop (адаптив)
- **Размер:** 1280px+
- **Навигация:** боковая или верхняя (нижняя панель на десктопе -- антипаттерн)
- **Контент:** может использовать дополнительное пространство (расширенные карточки, боковые панели, двухколоночные layout'ы), а не просто центрированная мобилка
- **Баттл на десктопе:** можно разместить ScoreBar и контент рядом, а не стопкой

Дизайнер решает, как именно контент адаптируется. Главное -- десктоп не должен выглядеть как растянутый мобильный экран.

### PWA
- Приложение работает в standalone режиме (без адресной строки браузера)
- Нужны иконки приложения: 192x192 и 512x512 PNG
- Splash screen (опционально)

### Реализация
- Стек: Next.js + Tailwind CSS
- Иконки: SVG
- Шрифты: Google Fonts или self-hosted

## 10. Состояния экранов

Помимо основного вида, дизайнер должен предусмотреть:

| Состояние | Где встречается | Что показать |
|-----------|----------------|--------------|
| **Loading** | Все экраны с данными | Скелетоны или индикатор загрузки |
| **Empty** | Главная (0 баттлов), Профиль (нет истории), Лидерборд (пусто), AI-чат (нет диалогов) | Мотивирующее сообщение + CTA |
| **Error** | Потеря соединения, ошибка сервера | Сообщение + кнопка повтора |

Не нужны отдельные фреймы на каждый -- достаточно одного примера каждого состояния в стайл-гайде, чтобы мы могли переиспользовать паттерн.

## 11. Что НЕ нужно

- Анимации в After Effects / Lottie (реализуем CSS-анимациями)
- Создание контента (тексты вопросов, описания модулей)

## 12. Формат сдачи

- **Figma-файл** с:
  - Стайл-гайд (цвета, типографика, компоненты) -- обе темы
  - Макеты всех экранов: mobile (375px) + desktop (1280px+) -- обе темы
  - Состояния интерактивных элементов (hover, active, disabled, error)
  - Все 5 фаз баттла (отдельные фреймы)
  - Примеры loading/empty/error состояний
- **Экспорт ассетов** (если есть кастомные иконки/иллюстрации): SVG
- **Иконки приложения**: 192x192 и 512x512 PNG

## 13. Приоритеты

1. **Высший:** Экран баттла (все 5 фаз) + выбор сложности -- это сердце продукта
2. **Высокий:** Главная + Профиль -- ежедневные экраны + мотивация
3. **Средний:** Поиск баттла + VS экран -- первое впечатление от боя
4. **Средний:** Обучение + Модуль + AI-наставник -- обучающий core loop
5. **Средний:** Разминка -- ежедневная активность
6. **Базовый:** Логин + Лидерборд + Навигация -- стандартные элементы
